## 📊 数据源

### 在线表格

**表格名称**：基金净值数据 - 可编辑版  
**存放位置**：飞书云盘 → 投资分析 文件夹

### 数据格式

| 列名 | 字段 | 示例 | 说明 |
|-----|------|------|------|
| A 列 | 估值日 | 2026/1/31 | 日期格式：YYYY/M/D |
| B 列 | 持仓简称 | Fund_A | 基金名称（与现有名称一致） |
| C 列 | 单位净值 | 12.9052 | 保留 4 位小数 |

---

## 🔄 标准操作流程（SOP）

### 步骤 1：在表格中添加数据

1. **打开表格**
   - 从飞书云盘 → 投资分析 文件夹 → 基金净值数据 - 可编辑版

2. **添加新数据**
   - 滚动到表格最后
   - 在最后一行下方添加新数据
   - 格式示例：
     ```
     2026/2/28,Fund_A,13.5000
     2026/2/28,Fund_B,14.2000
     ...
     ```

3. **检查数据**
   - ✅ 日期格式正确（2026/2/28）
   - ✅ 基金名称与现有名称一致
   - ✅ 净值保留 4 位小数
   - ✅ 所有基金都已添加

4. **确认保存**
   - 在线表格自动保存，无需手动保存

---

### 步骤 2：同步到服务

1. **导出 CSV**
   - 文件 → 下载为 → CSV

2. **上传到服务器**
   ```bash
   # 将 CSV 文件复制到服务器数据目录
   scp upload.csv user@server:/path/to/wmcapital/data/
   ```

3. **重启服务**
   ```bash
   cd /path/to/wmcapital
   ./manage.sh restart
   ```

---

### 步骤 3：验证更新结果

1. **访问网页**
   ```
   http://server-ip/
   ```

2. **检查项目**
   - ✅ 基金数量正确
   - ✅ 最新月份数据已显示
   - ✅ 图表数据已更新
   - ✅ 排名数据正确

3. **检查服务状态**（可选）
   ```bash
   ./manage.sh status
   ```

## ⚠️ 常见问题处理

### 问题 1：服务启动失败

**症状**：重启后服务未启动

**解决方法**：
```bash
# 1. 查看错误日志
./manage.sh logs

# 2. 检查数据文件格式
head -5 data/upload.csv

# 3. 重新导出 CSV（如格式错误）

# 4. 再次重启
./manage.sh restart
```

---

### 问题 2：网页数据未更新

**症状**：重启服务后网页仍显示旧数据

**解决方法**：
```bash
# 1. 清除浏览器缓存
# Ctrl+Shift+Delete → 清除缓存

# 2. 强制刷新网页
# Ctrl+F5

# 3. 检查服务状态
./manage.sh status

# 4. 再次重启
./manage.sh restart
```

---

### 问题 3：部分基金缺失

**症状**：网页显示的基金数量少于预期

**解决方法**：
```bash
# 1. 检查 CSV 中的基金名称
cut -d',' -f2 data/upload.csv | sort -u

# 2. 对比应包含的基金列表
# 检查是否有拼写错误

# 3. 修正基金名称
nano data/upload.csv

# 4. 重启服务
./manage.sh restart
```
